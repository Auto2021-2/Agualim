Comunicación SCADA


La comunicación entre los diferentes software se realiza usando la herramienta ignition, la cual permite conectar dispositivos que se comuniquen utilizando el protocolo OPC. Ademas esta aplicación permite conectar dispositivos que no se encuentran en la misma red local. La comunicación OPC de los programas Studio5000, NX siemens y RobotStudio se da a partir del acceso o la creacion de servidores OPC, los cuales se encargan de tomar datos que pueden estar en un formato nativo del fabricante y llevarlos al protocolo estandarizado llamado OLE for process control (OPC).



RobotStudio

El software RobotStudio no tiene la posibilidad de acceder a servidores OPC o crear uno para transmitir datos por si mismo. Para logralo primero se requiere una version de Robotware compatible con comunicaciones OPC, lo cual se traduce en que el controlador virtual tiene la capacidad de enviar y recibir datos OPC, para lograr la conexion con ignition se requiere usar el software IRC5 OPC UA, en donde se permite conectar los controladores que se estan ejecutando a un servidor OPC UA, el cual se puede conectar facilmente desde ignition.

NXSiemens

Para el caso de NX, el software tiene ya integrada la posibilidad de establecer comunicaciones OPC UA. Por esta razon se puede acceder al servidor OPC que crea ignition por defecto para conectar las variables de NX con las tag correspondientes dentro de la HDMI.

RSlink

Para este software tambien usa un server OPC UA para transmitir los datos a traves de la red, pero a diferencia de robotstudio se puede correr un servidor sin ningun otro software adicional. De esta manera desde ignition se crea una nueva conexión OPC.

Pagina de Inicio

En la pagina de inicio se debe elegir el producto que se va a procesar e iniciar el linea de proceso y posee unos indicadores para asegurar que ´hay pallets y canastas para realizar el proceso y tambien se verifica que no exita ningun tipo de alerta de seguridad en la celda robotizada. Ademas se tiene un pulsador de emergencia en caso que se deba detener toda la linea de producción.



Pagina de la celda robotica


En esta ventana se puede observar la cantidad de canastillas que se han usado para armar el pallet y la cantidad de pallets que ya han sido terminados. Por otro lado tambien permite claramente visualizar si alguna alarma de emergencia se enciende.



Pagina de empacado

En esta pagina tiene como objetivo la supervision de los actuadores y sensores de la linea de producción virtual, donde se puede observar el intercambio de señales entre Rslink y NX siemens, aunque se observan botones de 2 estados realemente la signacion de los valores de las tag entre NX y RSlink se da usando eventos que ejecutan un script para sobre escribir el tag homologo en el otro software, en otras palabras, si Nx cambia el estado de un sensor este activa un evento y un script para cambiar el tag correspondiente en RSlink.




Oportunidades de mejora

Durante el desarrollo del proyecto se tuvieron bastantes dificultades para correr todos los programas involucrados en las conexiones OPC en un mismo computador. Por este motivo se podria llevar ignition a la nube por medio de una maquina virtual como las ofrecidas por google. Con esta nueva arquitectura seria posible acceder a los servidores de ignition usando una ip correspondiente a la de la maquina virtual y solo con acceso a internet se podria conectar diferentes servidores OPC que no estan en la misma red local.
